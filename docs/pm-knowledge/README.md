# PM knowledge base — catalog

The certified-PM knowledge library for the `pm-craft` skill. Each file distills a framework
to the level a certified practitioner operates at, then ties it to **OpenProject** (see
`../openproject-concepts-and-pm.md`) and to delivery practice.

This catalog is **not a one-time setup** — it is grown continuously (see CLAUDE.md →
"Continuous learning"). When a framework, a certification syllabus, or OpenProject itself
changes, update the relevant file and cite the primary source.

## How to use this catalog

1. Before facilitating or designing a process, read the relevant framework file **and**
   the OpenProject mapping, so advice is grounded in both theory and the actual tool.
2. Prefer framework-accurate vocabulary with the user, but always translate to concrete
   OpenProject actions (work packages, versions, queries, boards, wiki).
3. When the user's context is ambiguous (which framework? scaled or single team?), ask —
   don't assume Scrum by default.
