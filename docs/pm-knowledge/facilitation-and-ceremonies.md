# Facilitation & ceremonies — reference

How to run the recurring events well, and the OpenProject artifacts each produces.

## Backlog refinement (ongoing, ~10% of capacity)
Goal: top of the backlog is ready (clear, estimated, small, INVEST, AC written). Pull from
the intake query; for each item: clarify value & scope, write **acceptance criteria**
(Gherkin), estimate (story points), set priority, link to epic, set type/category.
→ OpenProject: update WPs in the priority-sorted query; nothing left un-triaged.

## Sprint planning
Inputs: refined backlog, velocity, capacity. Set a **Sprint Goal**; pull a realistic set;
confirm each has owner, estimate, AC; agree the plan.
→ OpenProject: create/confirm the **Version** (dates) + goal (Version description/Wiki);
assign WPs to the version; set assignees/points.

## Daily standup (15 min)
Re-plan toward the goal; surface blockers (not a status report to a manager).
→ OpenProject: task board / a "current sprint" query; flag blocked WPs via relations.

## Sprint review
Inspect the Increment with stakeholders; collect feedback; adapt the backlog. A working
session, not a one-way demo.
→ OpenProject: close accepted WPs (AC verified); create follow-up WPs from feedback.

## Sprint retrospective
Inspect people/process/tools. Format e.g. went-well / didn't / actions, or Start-Stop-
Continue. Pick **a few** improvements and make them actionable + owned.
→ OpenProject: Wiki retro page + improvement **work packages** (so actions don't evaporate).

## Sprint close / carry-over
Verify AC, close done items, **move incomplete WPs to the next version**, note carry-over
reasons, check burndown/velocity.

## Kanban cadences (Scrumban)
Replenishment (refill the ready queue when WIP drops), delivery (release), flow review
(metrics), plus standup. Enforce **WIP limits** as policy.

## Facilitation principles
Timebox; make the goal explicit; ensure psychological safety (everyone speaks); decisions
recorded; one conversation at a time; end with owned actions. Tailor ceremonies to team
maturity — don't impose ritual for its own sake.
